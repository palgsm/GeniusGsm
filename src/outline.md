مشروع: GeniusGsm — مسودة أوتلاين

1) لمحة عامة
- الهدف: بناء منصة خدمات بريد مؤقت وأدوات مصاحبة تستهدف المستخدمين الذين يحتاجون لعناوين بريد مؤقتة وتكاملات مع أدوات أخرى.
- الجمهور المستهدف: مطورون، مختبرون، مستخدمو خدمات الويب الذين يريدون حماية الخصوصية، ومشاريع تحتاج إلى عناوين مؤقتة لاختبارات التسجيل.

2) نطاق النسخة الأولى (MVP) — ميزات ذات أولوية عالية
- إنشاء عنوان بريد مؤقت جديد عبر واجهة ويب.
- استقبال الرسائل وعرضها في صندوق وارد مؤقت (قابل للتحديث/تجديد لفترة قصيرة).
- API بسيط لإنشاء صندوق مؤقت واسترجاع الرسائل (REST API عبر Django REST Framework).
- إعدادات احتفاظ بالرسائل (قابل للتكوين: 1 ساعة، 24 ساعة، أو مخصص).

3) ميزات مرحلية (بعد الـMVP)
- تسجيل دخول/حسابات مستخدمين (اختياري): لحفظ الصناديق، إدارة المدة، لوحة تحكم.
- تكاملات (اختياري): Webhooks، تكامل مع خدمات طرف ثالث (Gmail API, SMS gateways، أدوات اختبار تلقائي).
- واجهة لوحة تحكم متقدمة + ملخص API وRate limiting.
- دعم إنشاء نطاقات فرعية أو نطاقات قابلة للتكوين.

4) اعتبارات تقنية
- إطار العمل الخلفي: Django + Django REST Framework.
- قاعدة البيانات: PostgreSQL.
- مهام خلفية: لا نستخدم Celery؛ سنعتمد مهام مجدولة بسيطة (cron أو `Django` management commands) أو حلول خفيفة عند الحاجة لمعالجة الحذف المؤقت وعمليات الصيانة.
- التخزين المؤقت: Redis.
- استضافة/بيئة: Docker + docker-compose (مقترح للبدء).

5) خصوصية واحتفاظ بالبيانات
- الافتراض: الرسائل المؤقتة تُحذف تلقائياً بعد فترة قابلة للتعديل.
- دعم خيارات الامتثال الأساسية (تمكين/تعطيل تسجيل الـIP، سياسة حذف بيانات المستخدم عند الطلب).

6) بنية المشروع مقترحة (مستويات)
- جميع التطبيقات والـDjango project داخل مجلد `src/` على النحو التالي:
	- `src/project/` — إعداد Django الرئيسي (settings, urls, wsgi/asgi).
	- `src/mailbox/` — منطق إنشاء الصناديق واستقبال الرسائل.
	- `src/api/` — واجهات REST (DRF).
	- `src/users/` — إدارة المستخدمين (اختياري).
	- `src/docker/`، `src/compose/` — ملفات تطوير محلية وملفات تكوين الحاويات.
	- ملف `manage.py` يبقى في جذر المشروع أو يمكن نقله إلى `src/` حسب تفضيل النشر.

7) خطوات تالية مقترحة
- تأكيد 2-3 ميزات أساسية للـMVP.
- قرار حول تسجيل المستخدمين أم لا.
- إعداد بيئة تطوير محلية (Docker) وملف `requirements.txt`/`pyproject.toml`.

أسئلة قصيرة لك الآن:
- أي 3 ميزات تضعها كأولوية للـMVP؟
- هل نفعّل تسجيل المستخدمين في البداية؟
- هل تفضّل واجهة واجهات مبدئية بـDjango templates أم React/SPA لاحقاً؟
